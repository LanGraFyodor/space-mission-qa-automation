# Space Mission Test Automation

В этом репозитории собраны 3 проекта автоматизированного тестирования (API, UI, Load) и 1 проект основного веб-сервиса (API Бэкенд). 
Все проекты написаны на **Java 17** и используют сборщик **Maven**.

---

##  0. Основной проект (Бэкенд)
Локальный REST API сервер (Space Mission Management) со встроенной in-memory БД (H2) и JWT-авторизацией. Он необходим для работы API и нагрузочных тестов.

**Что проделано:**
* С нуля написан микросервис на Spring Boot (модели: Миссии, Группировки, Спутники).
* Настроена JWT аутентификация для защиты эндпоинтов.
* Реализованы полные CRUD-операции для всех сущностей (17 ручек).

**Как запустить:**
1. Откройте терминал в папке `demo-api`.
2. Выполните команду:
   ```bash
   mvn spring-boot:run
   ```
3. Сервер будет запущен на `http://localhost:8080`. **Не закрывайте этот терминал, пока тестируете API и нагрузку!**

*(Данные для входа: любой логин, пароль `password`)*

---

##  1. Задача 1: API Автотесты
Проект покрывает 100% эндпоинтов основного сервера `demo-api`. Использует JUnit 5, RestAssured и Allure.

**Что проделано:**
* Написано **27 тестов**, покрывающих абсолютно все API-ручки (GET, POST, PUT, DELETE).
* Реализована автоматическая передача JWT-токенов в запросах.
* Подключен Allure (с аннотациями `@Severity`, `@Story`).

**Как запустить:**
1. Убедитесь, что сервер `demo-api` запущен.
2. Откройте терминал в папке `api-autotests`.
3. Запустите тесты:
   ```bash
   mvn clean test
   ```
4. **Сгенерировать Allure-отчёт:**
   ```bash
   mvn allure:serve
   ```

---

##  2. Задача 2: UI Автотесты
UI-тесты для сайта `http://uitestingplayground.com`. Использует Selenium WebDriver, WebDriverManager, JUnit 5 и Allure.

**Что проделано:**
* Успешно автоматизировано **5 сценариев**: `Sample App` (логин), `Shadow DOM` (работа с теневым деревом), `Dynamic ID` (элементы с меняющимся ID), `Click` (игнорирование DOM) и `Text Input`.
* *Примечание: Задание Frames заменено на Shadow DOM из-за отсутствия актуального стенда.*
* Добавлено автоматическое прикрепление скриншотов при падении.

**Как запустить:**
1. Откройте терминал в папке `ui-autotests`.
2. Запустите тесты (рекомендуется Headless-режим без окон):
   ```bash
   mvn clean test -Dheadless=true
   ```
3. **Сгенерировать Allure-отчёт:**
   ```bash
   mvn allure:serve
   ```

## 3. Задача 3: Нагрузочные тесты
Сценарий симулирует работу реальных операторов системы. Использует **Gatling** (Java DSL).

**Что проделано:**
* Написан комплексный бизнес-сценарий (вход, создание миссий/групп/спутников, привязка, получение списков и удаление).
* Настроен реалистичный комбинированный профиль: разгон до **50 параллельных пользователей**, удержание нагрузки (**60+ секунд**).
* Обеспечена автоматическая генерация графического отчёта.

**Как запустить:**
1. Убедитесь, что сервер `demo-api` запущен.
2. Откройте терминал в папке `load-tests`.
3. Запустите симуляцию:
   ```bash
   mvn gatling:test
   ```
4. **Просмотр отчёта:** Gatling автоматически сгенерирует HTML-отчёт. В конце работы скрипта в консоли появится путь к файлу `index.html`. Скопируйте путь и откройте его в браузере.
